Vitaes uses [Zod](https://zod.dev) for runtime type validation. All types are defined in `@vitaes/types/resume`.

## Core Types

### ResumeSchema

The main schema for a complete resume document.

```typescript packages/types/src/resume.ts
const ResumeInputSchema = z.object({
  config: ResumeConfigInputSchema,
  personalInfo: PersonalInfoSchema,
  sections: z.array(SectionSchema),
})

export const ResumeSchema = ResumeInputSchema.transform((data) => ({
  ...data,
  config: {
    ...data.config,
    footerLeft: data.config.footerLeft ?? defaultFooterOption,
    footerCenter: data.config.footerCenter ?? defaultFooterOption,
    footerRight: data.config.footerRight ?? defaultFooterRight,
  },
}))

export type IResume = z.infer<typeof ResumeSchema>
```

<Expandable title="Type Definition">
  <ResponseField name="config" type="ResumeConfig">
    Resume configuration including template, colors, and formatting options
  </ResponseField>
  
  <ResponseField name="personalInfo" type="PersonalInfo">
    Personal information including name, position, contact details, and photo
  </ResponseField>
  
  <ResponseField name="sections" type="Section[]">
    Array of content sections (text, timeline, list, or taxonomy)
  </ResponseField>
</Expandable>

### ResumeValidationSchema

```typescript packages/types/src/resume.ts
export const ResumeValidationSchema = z.object({
```

## Configuration

### ResumeConfigSchema

Configuration for resume appearance and layout.

```typescript packages/types/src/resume.ts
const ResumeConfigInputSchema = z.object({
  template: TemplateSchema.optional(),
  themeColor: AwesomeColorSchema,
  headerAlign: z.enum(['left', 'center', 'right']),
  sectionColorHighlight: z.boolean(),
  fontSize: z.number().positive(),
  pageSize: z.enum(['A4', 'LETTER']),
  footerLeft: FooterOptionSchema.optional(),
  footerCenter: FooterOptionSchema.optional(),
  footerRight: FooterOptionSchema.optional(),
})

export const ResumeConfigSchema = ResumeConfigInputSchema.transform((data) => ({
  ...data,
  template: data.template ?? 'awesome',
  footerLeft: data.footerLeft ?? defaultFooterOption,
  footerCenter: data.footerCenter ?? defaultFooterOption,
  footerRight: data.footerRight ?? defaultFooterRight,
}))

export type ResumeConfig = z.infer<typeof ResumeConfigSchema>
```

<Expandable title="Fields">
  <ParamField path="template" type="Template">
    Resume template name. Defaults to `awesome`.
  </ParamField>
  
  <ParamField path="themeColor" type="AwesomeColor" required>
    Theme color for the resume
  </ParamField>
  
  <ParamField path="headerAlign" type="'left' | 'center' | 'right'" required>
    Header alignment
  </ParamField>
  
  <ParamField path="sectionColorHighlight" type="boolean" required>
    Whether to highlight section titles with theme color
  </ParamField>
  
  <ParamField path="fontSize" type="number" required>
    Base font size (must be positive)
  </ParamField>
  
  <ParamField path="pageSize" type="'A4' | 'LETTER'" required>
    Paper size for PDF export
  </ParamField>
  
  <ParamField path="footerLeft" type="FooterOption">
    Left footer configuration
  </ParamField>
  
  <ParamField path="footerCenter" type="FooterOption">
    Center footer configuration
  </ParamField>
  
  <ParamField path="footerRight" type="FooterOption">
    Right footer configuration (defaults to showing page numbers)
  </ParamField>
</Expandable>

## Section Types

Resumes are composed of sections. Each section has a type that determines its structure.

### SectionSchema

Discriminated union of all section types.

```typescript packages/types/src/resume.ts
export const SectionTypeSchema = z.enum([
  'text',
  'timeline',
  'list',
  'taxonomy',
])

export const SectionSchema = z.discriminatedUnion('type', [
  TextSectionSchema,
  TimelineSectionSchema,
  ListSectionSchema,
  TaxonomySectionSchema,
])

export type SectionType = z.infer<typeof SectionTypeSchema>
export type Section = z.infer<typeof SectionSchema>
```

### BaseSectionSchema

Common fields for all sections.

```typescript packages/types/src/resume.ts
export const BaseSectionSchema = z.object({
  id: z.string(),
  title: z.string(),
  type: SectionTypeSchema,
})
```

<Expandable title="Fields">
  <ParamField path="id" type="string" required>
    Unique identifier for this section
  </ParamField>
  
  <ParamField path="title" type="string" required>
    Section title displayed in the resume
  </ParamField>
  
  <ParamField path="type" type="SectionType" required>
    Section type (text, timeline, list, or taxonomy)
  </ParamField>
</Expandable>

## Type Guards

Helper functions to narrow section types.

```typescript packages/types/src/resume.ts
export const isTextSection = (section: Section): section is TextSection =>
  section.type === 'text'

export const isTimelineSection = (
  section: Section,
): section is TimelineSection => section.type === 'timeline'

export const isListSection = (section: Section): section is ListSection =>
  section.type === 'list'

export const isTaxonomySection = (
  section: Section,
): section is TaxonomySection => section.type === 'taxonomy'
```

**Usage:**

```typescript
import { isTimelineSection } from '@vitaes/types/resume'

for (const section of resume.sections) {
  if (isTimelineSection(section)) {
    // TypeScript knows section is TimelineSection
    section.entries.forEach(entry => {
      console.log(entry.position, entry.title)
    })
  }
}
```
